§ 01 — endpoint

One endpoint. Ask for exactly what you need.

?include= selects which sections the agent returns. Default gives you the essentials; all gives you everything including processes, disks, and Docker containers. Toggle below.

include 5 of 10
GET 20012ms
§ 02 — surface area

Ten sections. Five on by default. No magic.

host default hostname, OS, platform, kernel, uptime, boot time, process count string · 7 fields
cpu default model, core count, per-core and total usage percentages object · per-core
memory default total, used, available — as bytes and human-readable strings bytes + human
load default 1, 5, 15-minute load averages 3 floats
network default interfaces with MAC, IPs, flags, MTU, plus I/O counters array · live
swap opt-in total, used, free — bytes and human-readable, plus usage % ?include=swap
disks opt-in mounted partitions with device, mountpoint, fstype, sizes ?include=disks
processes opt-in top 50 by CPU — PID, name, status, CPU%, mem%, user, cmdline ?include=processes
users opt-in logged-in users with terminal and host ?include=users
docker opt-in server version plus all containers (name, image, state, ports, labels) ?include=docker
§ 03 — auth

Opt-in bearer token. Nothing else in the way.

Set TOKEN and every request requires it. Leave it unset for open access on trusted networks. Both header and query-param styles are supported.

— set the token
shellenv
TOKEN=mysecret ./machine-agent
— header (preferred)
requestbearer
curl -H "Authorization: Bearer mysecret" \
     http://192.168.1.10:7891/metadata
— query param (convenience)
request?token
curl "http://192.168.1.10:7891/metadata?token=mysecret"
— unauthenticated
response401
// without Authorization header or ?token=
HTTP/1.1 401 Unauthorized
{ "error": "missing or invalid token" }
note
Health check at GET /health is always open. It returns {"status":"ok"}.
§ 04 — discovery

Agents announce themselves. You don't keep a list.

Each running agent advertises as _machine-agent._tcp over mDNS / Bonjour. Your central tool finds every agent on the LAN without knowing a single IP up front.

— discover
shelldiscover
./machine-agent -discover
— opt out
shellno-mdns
./machine-agent -no-mdns
# don't announce on the LAN
In Docker, mDNS needs --net=host. With bridge networking the announcement stays inside the container — fine if you don't want broadcast.

§ 05 — deploy

Docker, binary, or source. Pick one.

A — docker
Pull and run. Mount the Docker socket to collect container info from the host.
shpull
docker pull klipitkas/machine-agent
docker run -d -p 7891:7891 \
  -v /var/run/docker.sock:\
/var/run/docker.sock:ro \
  klipitkas/machine-agent
B — binary
Build from source — single static binary, around 4 MB, no runtime deps.
shbuild
go build -o machine-agent \
  ./cmd/machine-agent

./machine-agent
./machine-agent -port 9999
./machine-agent -version
C — host metrics (linux)
Report the host's metadata instead of the container's — mount /proc, /sys, /etc.
sh--net=host
docker run -d --net=host --pid=host \
  -v /proc:/host/proc:ro \
  -v /sys:/host/sys:ro \
  -v /etc:/host/etc:ro \
  -e HOST_PROC=/host/proc \
  -e HOST_SYS=/host/sys \
  -e HOST_ETC=/host/etc \
  machine-agent
— repository layout
treemachine-agent/
machine-agent/
├── cmd/machine-agent/main.go          # entry point
├── internal/
│   ├── collector/
│   │   ├── collector.go          # metadata collection
│   │   └── types.go              # data types + helpers
│   ├── discovery/
│   │   └── mdns.go               # mDNS advertisement
│   └── server/
│       └── server.go             # HTTP, auth, routing
├── go.mod
└── README.md

Four packages, zero ceremony. The collector does the work, the server exposes it, discovery announces it, main.go wires them together.

If you want to add a metadata section, you touch collector/. If you want to change transport, you touch server/. That's it.

Drop it on every box. Query one place. Move on.
